# Code-Comment Generator

A neural sequence-to-sequence model that generates natural-language comments for Java methods, deployed as a Visual Studio Code extension.

Highlight a Java method, run the command, and a Javadoc-style comment is inserted above your selection — generated by a fine-tuned CodeT5 model served from a local or Colab-hosted backend.

```java
/**
 * Returns the user with the given identifier, or null if no such user exists.
 */
public User findUserById(long id) {
    return repository.findById(id).orElse(null);
}
```

## Project overview

The project implements two distinct seq2seq approaches and compares them on the same held-out test set:

**Approach A — Transformer from scratch.** A standard encoder–decoder Transformer (3 layers, 4 attention heads, 256 hidden dimensions, ~5M parameters) implemented in modern PyTorch. The architecture is built component-by-component (positional encoding, multi-head attention, encoder block, decoder block) rather than buried under a library abstraction, making the mechanics of attention and masking explicit in the code.

**Approach B — Fine-tuned CodeT5.** Salesforce's `codet5-small` (60M parameters), pre-trained on ~8M code/text pairs across multiple languages, fine-tuned for Java code summarization. This is the production-quality model that powers the VS Code extension.

Both approaches share the same data pipeline, the same evaluation, and the same downstream inference server, so the comparison between them is direct.

## Dataset

[CodeSearchNet](https://github.com/github/CodeSearchNet) — a benchmark dataset of paired (code, docstring) examples mined from open-source GitHub repositories. The Java subset contains roughly 500K function/docstring pairs after the official train/valid/test split, drawn from real production codebases.

The notebook downloads the dataset automatically.

## Methods

**Preprocessing.** Each docstring is reduced to its first sentence (Javadoc summaries before the `@param` block). Non-ASCII docstrings, HTML tags, exact duplicates, and pairs where the docstring is longer than the code are dropped. Code is filtered to functions between 10 and 400 words; docstrings to 3–50 words.

**Tokenization.** The from-scratch Transformer uses two separate Byte-Pair Encoding tokenizers (8K vocab each) — one for code, one for docstrings — trained from scratch on the cleaned corpus. CodeT5 uses its native pre-trained Roberta tokenizer.

**Training the from-scratch Transformer.** AdamW with one-cycle learning rate scheduling, cross-entropy loss with label smoothing (0.1) and `<pad>` ignored, gradient clipping at 1.0. Trained for 5 epochs on 20K subsampled training pairs at batch size 32.

**Fine-tuning CodeT5.** Hugging Face `Seq2SeqTrainer` with 3 epochs on 30K training pairs, learning rate 5e-5, warmup of 500 steps, weight decay 0.01, FP16 mixed precision, batch size 16.

**Evaluation.** Both models are evaluated on 500 held-out test examples using:
- **BLEU-4** (Papineni et al., 2002) — n-gram precision against reference, the standard machine-translation metric.
- **ROUGE-L** (Lin, 2004) — longest common subsequence F1, more forgiving on word order.

A qualitative side-by-side of 8 random examples is included in the notebook to complement the metrics, since BLEU and ROUGE are imperfect proxies for whether a comment is actually useful.

## Running it yourself

### 1. Train the models

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/YOUR-USERNAME/code-comment-generator/blob/main/code_comment_generator.ipynb)

Open the notebook in Colab, set runtime to GPU (T4 is sufficient), and run all cells. Approximate runtime on a free T4:

- Data download and exploration: ~5 min
- From-scratch Transformer training (5 epochs): ~15 min
- CodeT5 fine-tuning (3 epochs): ~30 min
- Evaluation: ~5 min
- **Total: ~55 min**

The notebook saves the fine-tuned CodeT5 to `codet5-finetuned/` for serving.

### 2. Serve the model

```bash
pip install -r requirements.txt
python server.py
```

This exposes `POST /generate` on port 8000. To make it reachable from a remote VS Code instance (e.g., serving from Colab), the notebook includes a deployment cell that wraps the same FastAPI app and exposes it via ngrok.

### 3. Install the VS Code extension

```bash
npm install
```

Open the folder in VS Code, press F5 to launch a development host. In the host window:

1. Open Settings → Extensions → Code-Comment Generator
2. Set "Server URL" to your server's address (default: `http://localhost:8000/generate`; with ngrok: `https://xxxx.ngrok.io/generate`)
3. Open any Java file, highlight a method, right-click → *Code Commenter: Generate Comment for Selection*

## Limitations

A few things worth being upfront about:

- **The from-scratch Transformer is not a research contribution.** It's a clean implementation of "Attention Is All You Need" (Vaswani et al., 2017). It's included to demonstrate understanding of the underlying architecture, not to claim novelty.
- **CodeT5 does most of the work.** The fine-tuning helps meaningfully on Java summarization, but the heavy lifting is the pre-training that Salesforce performed in 2021.
- **Both models hallucinate.** Neither model executes the code or reasons about its semantics — they're pattern-matching against training distributions. A method named `deleteUser` that actually creates a user would still get summarized as "delete a user." This is a fundamental limit of the seq2seq approach for code.
- **Modern frontier LLMs outperform this approach zero-shot.** GPT-4, Claude, and Code Llama all produce higher-quality comments without any fine-tuning. The honest reason this project is worth building is to demonstrate the underlying machinery, not to compete with frontier models on output quality.

## What I'd explore next

- **Compare against zero-shot LLM prompting** — likely better quality, worse latency and cost; quantifying the trade-off is interesting.
- **Identifier-aware data augmentation** — synonym substitution on variable names to test whether the model genuinely understands code structure or just memorizes training patterns.
- **Human evaluation** — programmers rating comments on usefulness, accuracy, and conciseness on a Likert scale. A small N=50 study would say more than another decimal place of BLEU.
- **Multilingual extension** — CodeT5 is multilingual; the same pipeline could fine-tune on the full CodeSearchNet (Java, Python, JavaScript, Go, Ruby, PHP).
